# File Upload #
Eazy
- Upload payload.php
- Access http://localhost:4280/vulnerabilities/upload/../../hackable/uploads/payload.php?cmd=cat /etc/passwd

Medium
- Burpsuite set intercept on
- Upload payload.php
- Intercepted post change Content-Type: application/x-php to Content-Type: image/jpeg
- For testing purposes also change filename="payload.php" to filename="payload123.php"
- Forward Request
- Access http://localhost:4280/vulnerabilities/upload/../../hackable/uploads/payload123.php?cmd=cat /etc/passwd

Hard
- Burpsuite set intercept on
- Upload any image say dvwa_email.png
- Intercepted post and change the content of png file from

# Stored XSS #
All payloads are similar from the reflected XSS, but as the dificulty is higher, xss vulnerability only can be exploited on the name parameter. On the browser the name parameter is limited 10 characters, so if we want to input more change the limit from 10 to something higher by inspecting with chrome tools/firefox tools.
Other method is by using burpsuite and changing the request parameters directly via intercept.
